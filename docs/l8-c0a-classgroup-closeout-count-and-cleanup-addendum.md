## 2. Reference Major Count Reconciliation

### Scope Difference Between L8-A and L8-C0

L8-A and L8-C0 use **different scopes** when counting reference majors from `学院专业数据库.xlsx`:

| Scope | Source | Count | Description |
|-------|--------|-------|-------------|
| L8-A union scope | Sheet 1 + Sheet 3 (both scanned into single `majorSet`) | **38** | Union of all unique major names across both sheets |
| L8-C0 sheet-1-only | Sheet 1 (学院专业数据库) only | **34** | Unique major names in the official major list |
| Sheet 3 class-sheet scope | Sheet 3 (班级数据库) only | **35** | Unique major names in the per-class data |

### Root Cause of 38 vs 34

L8-A's `readMajorXlsx()` scans **both** sheets (`学院专业数据库` and `班级数据库`) and adds every `专业` value to a single `majorSet`. This produces a union of 38 unique major names.

Sheet 3 (班级数据库) has 35 unique major names. Of these, 4 exist in sheet 3 but NOT in sheet 1:

- 4 majors appear only in class data, with slightly different naming from sheet 1 or with no corresponding sheet 1 entry

Sheet 1 has 34 unique major names. 3 sheet-1 majors have no class rows in sheet 3 (majors defined but not yet assigned classes).

34 (sheet 1) + 4 (sheet-3-only) = 38 = L8-A union count.

### Correct Counts

| Metric | Value |
|--------|-------|
| Raw major-like rows (sheet 1) | 34 |
| Distinct normalized major keys (sheet 1) | 34 |
| Distinct major+direction keys (sheet 1) | 34 |
| Distinct major keys with at least one class (sheet 3) | 35 |
| Distinct major keys WITHOUT any class (sheet 1 only) | 3 |
| Sheet-level major count (sheet 1) | 34 |
| Sheet-level major count (sheet 3) | 35 |
| Union major count (L8-A scope) | 38 |
| Reference class count (sheet 3) | **227** (stable, unchanged) |

### Canonical Scope Decision

For ClassGroup reconciliation, the canonical reference source is **sheet 1 (学院专业数据库) = 34 unique majors**. This is the official major list. Sheet 3 per-class data is supplementary. The 4 sheet-3-only majors are naming variants or unlisted programs.
